# 项目当前进度文档

更新日期：2026-03-24

## 一、项目目标

本项目为一个即时通信 Android 客户端，需求包括：

1. 基于 SIP 的音视频通话
2. 文字、图片、语音、视频的发送和接收
3. 群聊与多方音视频通话
4. 后台管理与查询统计
5. 与实践 1 的 PC 客户端互联互通

目前项目按照四人分工推进：

1. 后端
2. 前端 UI
3. 文字传输
4. 音视频通话

---

## 二、总体进度概述

当前整体状态可以概括为：

- Android 客户端基础工程已搭建完成，项目可以正常编译运行
- 前端 UI 已从第一版静态展示推进到可交付后端联调的完整壳
- 登录页、工作区顶部信息条、底部导航、四大主页面已统一成同一套产品化视觉语言
- 本地假数据和本地模拟消息链路可以支撑消息、联系人、通话、后台管理的整体演示
- 真实后端、真实文字传输、真实 SIP 音视频通话仍未落地

当前最明显的进展仍然集中在前端 UI；但与 3 月中旬相比，前端已经不再只是“演示界面”，而是具备了明确的接口预留、交付清单和联调语义。

## 三、按分工的当前进度

### 1. 后端

当前状态：未开始

已完成内容：

- 暂无真实服务端代码
- 暂无数据库设计
- 暂无可用接口实现
- 暂无消息转发、用户管理、群聊管理、统计分析等服务落地

前端已预留、等待后端对接的内容：

- 用户登录与身份认证
- 会话列表与消息历史
- 文本 / 图片 / 语音 / 视频消息上传下载接口

前端已整理出的建议接口范围：

1. `POST /auth/login`
2. `GET /conversations`
3. `POST /messages/send`
4. `PATCH /groups/{groupId}/members`
5. `GET /calls/bootstrap`
6. `GET /admin/dashboard`

后续建议优先任务：

1. 先确定统一的数据表、协议字段和错误码
2. 优先完成登录、会话列表、消息发送接收接口

4. 最后联动 SIP 服务与 PC 端互通

当前评价：

- 进度：0%

- 说明：前端壳已经准备完毕，后端若继续空缺，会直接卡住后续真实联调

- 完成统计卡片、状态芯片、接口卡片、时间线、空状态等复用组件
- 完成后端交付检查项、接口清单、联调动态等状态模型接入

当前可直接演示的界面能力：

- 中文首页进入联调工作台
- 顶部工作区信息栏展示当前用户、SIP、环境、阶段
- 底部导航切换消息、联系人、通话、管理四个主域
- 消息页展示文本、图片、语音、视频四类消息卡片
- 消息页展示群聊路由键、交付检查项、后端接口接入重点
- 联系人页展示标签、状态、建议动作和响应窗口
- 通话页展示拨号中、来电、通话中、会议准备四类页面壳
- 管理页展示后端接口清单、互通协议字段和分级告警

当前不足：

- 登录页仍为演示入口，未接真实账号密码鉴权
- 群聊创建页、群聊详情页、成员管理页尚未独立拆页
- 通话中的独立全屏控制页尚未做成真实交互状态机
- 弱网态、鉴权失败态、无网络态、权限拒绝态仍不完整
- 深色模式下的专项打磨尚未做细

2. 补登录失败、接口失败、断网、权限拒绝等异常态
3. 将消息页的媒体卡片接入真实文件 URL 与上传进度
4. 将通话页的壳接入真实信令和媒体状态流

当前评价：

- 风险：低
- 说明：UI 已经可以支撑课程展示，也已经具备交付后端联调的结构完整性
